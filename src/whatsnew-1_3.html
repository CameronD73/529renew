Changes between versions 1.2.x and 1.3 of 529Renew.

The page appears only when 529renew is upgraded from version 1.2.x to 1.3

This significant change was required in September 2023, when 23andme changed the "Relatives in Common" table: they removed the "yes" or "no" indication whether DNA relatives in common shared an overlapping DNA segment and replaced them by a single "Compare" button. Now, 529renew code must examine the DNA segments between all matches, which will slow the acquisition progress even further.

To help this slightly, I have adapted the meaning of the minimum shared DNA (pct) option that previously applied only when you chose to record non-overlapping matches. This will now apply to all matches (because the code can no longer tell beforehand whether there is any DNA overlap), so you should ensure that it is not set at a value that is too high.

Implications for GDAT

Roberta Estes wrote that, in 2021, 23andme had already changed the way the overlaps in shared matches were reported. They were no longer true triangulations, matching each of 3 ways. 529 and You (and 529renew) also recorded the third pair of matching segments, which would permit confirmation of the full triangulation, but as far as I can tell this has never been done within the code.

529 does not save or report any triangulation. It does not even directly save the fact that the three testers are in common with each other. It merely records matching segments, exports them, and leaves it up to GDAT to infer triangulation or ICW. This is done (as far as I can tell) by identifying the 3-way segment matching.
